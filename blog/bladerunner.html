Bladerunner: Build a static website on Github with Blade and gulp.js

Meh, why not using Jekyll ? Because. All my latest projects are built with Laravel but it may be a bit overkill for a website build with HTML, CSS and JavaScript only. The goal is to get something really really light that can help me build a website with one HTML layout without copying it (DRY) on every pages. Here's come the idea, why not build something with Blade (the template engine from Laravel) and [gulp.js|(http://gulpjs.com) to compile SASS and Javascript.

Setupping the Github repository

I use a subdirectory called dist on the master branch to be the root directory of the repository's gh-pages branch. Within the master, we'll get all the stuff that we need to generate the static website and a cleaned version into dist subdirectory that will be use to push into gh-pages branch.

Blade (the template engine)

Blade is a simple, yet powerful templating engine provided with Laravel. Blade is driven by template inheritance and sections. So it's perfect for layout...

To use Blade as a standalone component, I installed Laravel-Blade from PhiloNL.

gulp.js (the streaming build system)

gulp.js make it easy to compile SASS and uglify JavaScript. But what I like the most about it is the ability to automatically run task in background on files changes with gulp.watch().

The final result: Bladerunner

With a little bit of magic (not really) of a PHP script to generate pages from Blade views and gulp.js, I have create Bladerunner. Clone it and use it on your own.
